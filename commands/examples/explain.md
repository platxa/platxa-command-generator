# Explain Code

Read the indicated file or function and provide a clear explanation of what it does.

## Guardrails

- Read the entire file before explaining (don't assume from names alone)
- Explain at the appropriate level for the code's complexity

## Instructions

1. Read the target code thoroughly
2. Identify the main purpose and key logic paths
3. Explain what the code does in plain language
4. Note any edge cases, error handling, or non-obvious behavior

## Output

Provide a concise explanation with:
- One-sentence summary of purpose
- Key logic walkthrough
- Notable patterns or potential issues
